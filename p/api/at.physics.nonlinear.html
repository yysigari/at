chromaticity(ring, method='linopt', dpm=0.02, npoints=11, order=3, dp=0, **kwargs)[source]#

Computes the non-linear chromaticities

This function computes the tunes for the specified momentum offsets. Then it fits this data and returns the chromaticity up to the given order (npoints>order)

Parameters:

  • ring (Lattice) – Lattice description

  • method (Optional[str]) –

    'linopt' (dfault) returns the tunes from the linopt6() function,

    'fft' tracks a single particle and computes the tunes with fft,

    'laskar' tracks a single particle and computes the tunes with NAFF.

  • dpm (Optional[float]) – Maximum momentum deviation

  • npoints (Optional[int]) – Number of momentum deviations

  • order (Optional[int]) – Order of the fit

  • dp (Optional[float]) – Central momentum deviation

Returns:

(fitx, fity), dpa, qz

Computes the tunes for a sequence of amplitudes

This function uses tunes_vs_amp() to compute the tunes for the specified amplitudes. Then it fits this data and returns result for dQx/dx, dQy/dx, dQx/dy, dQy/dy

Parameters:

  • ring (Lattice) – Lattice description

  • xm (Optional[float]) – Maximum x amplitude

  • ym (Optional[float]) – Maximum y amplitude

  • npoints (Optional[int]) – Number of points in each plane

  • window (Optional[int]) –

  • nturns (Optional[int]) – Number of turns for tracking

Returns:

  • q1 (ndarray)

  • x (ndarray) – x amplitudes

  • q_dx (ndarray)

  • y (ndarray) – y amplitudes

  • q_dy (ndarray)
